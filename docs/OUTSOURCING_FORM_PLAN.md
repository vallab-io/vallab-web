# 외주 개발 신청 폼 – 계획 (DB 없이)

## 목표

- 사이트에서 **외주 개발 신청**을 받을 수 있는 폼 제공
- **DB 없이** 운영 (인프라·유지보수 최소화)

---

## DB 없이 데이터 받는 방법 정리

| 방식 | 설명 | 장점 | 단점 |
|------|------|------|------|
| **1. 이메일 전송** | 폼 제출 시 API가 이메일로 전송 | 우리 서버만 사용, 데이터 우리 메일함에만 존재 | 이메일 API 키 필요, 스팸/도메인 설정 고려 |
| **2. 폼 서비스 (Formspree, Tally 등)** | 외부 서비스가 폼 받고 이메일/대시보드 제공 | 구현 빠름, 스팸/캡차 대응 용이 | 서비스 의존, 무료 한도 있음 |
| **3. Google 폼 / 시트** | Google Form 임베드 또는 시트 API로 저장 | 무료, 시트로 관리 가능 | 디자인 제약, 우리 사이트와 톤 통일 어려움 |
| **4. Notion** | Notion API로 DB 대신 페이지/테이블에 저장 | 구조화된 보기, 검색·필터 가능 | Notion API 키, 약간의 연동 개발 |
| **5. 슬랙/디스코드** | 제출 시 웹훅으로 메시지 전송 | 실시간 알림, 팀 공유 쉬움 | 장기 보관·검색에는 부적합 |

## 추천 방향

### 1안: **이메일 전송 (우리 서버만 사용)** ⭐ 추천

- **흐름**:  
  `폼 제출 → Next.js API Route → 이메일 발송(Resend/SendGrid 등) → jihoi.kang@vallab.io 등 수신`
- **장점**: DB 없음, 우리 도메인·브랜딩 유지, 데이터가 우리 메일함에만 쌓임, 나중에 메일함 검색/라벨로 관리 가능.
- **필요**: 이메일 발송 서비스 가입 + API 키, 환경변수로 키 관리.

**적합한 서비스 예시**

- [Resend](https://resend.com) – 개발자 친화적, 무료 티어 넉넉함 (월 3,000통 등)
- [SendGrid](https://sendgrid.com) – 무료 티어 있음
- [Nodemailer + Gmail SMTP](https://nodemailer.com) – 별도 서비스 없이 가능 (Gmail 보안 설정 필요)

### 2안: **Formspree / Tally (가장 빠르게)**

- **흐름**:  
  폼 UI만 우리 사이트에 두고, `action`을 Formspree/Tally URL로 보내거나 iframe/위젯으로 삽입.
- **장점**: 서버 코드·이메일 API 없이 가능, 스팸·봇 방지 옵션 있음, 제출 건은 이메일 + 서비스 대시보드에서 확인.
- **단점**: 무료 한도, 디자인/필드 제한 있을 수 있음.

## 구현 단계 (1안: 이메일 전송 기준)

1. **폼 페이지**
   - 예: `/contact` 또는 `/outsourcing` (외주 신청 전용).
   - 필드 제안:
     - 이름 / 회사명(선택)
     - 이메일
     - 연락처(선택)
     - 프로젝트 개요 (한 줄 또는 짧은 설명)
     - 상세 요청 사항 (textarea)
     - 예산/기한(선택) – 텍스트 또는 선택지
     - 참고 링크(선택)
   - 클라이언트: `react-hook-form` + `zod` 검증, shadcn/ui Input/Textarea/Button 사용.

2. **API Route**
   - `POST /api/outsourcing` (또는 `/api/contact`)에서:
     - body 파싱 및 zod로 검증
     - 스팸 방지: reCAPTCHA 또는 간단한 honeypot 필드
     - Resend/SendGrid 등으로 메일 전송 (제목/본문에 폼 값 포맷팅)
   - 성공 시 `200` + `{ success: true }`, 실패 시 `4xx/5xx` + 메시지.

3. **환경 변수**
   - `RESEND_API_KEY` (또는 사용할 서비스의 API 키) – Vercel 등에 등록.

4. **이메일 내용**
   - 제목: `[Vallab 외주신청] {이름} - {프로젝트 한 줄 요약}`
   - 본문: 이름, 이메일, 연락처, 개요, 상세, 예산/기한, 링크를 읽기 쉽게 정리 (HTML 또는 플레인 텍스트).

5. **테스트**
   - 로컬에서 제출 → 실제 수신 메일 확인.
   - Vercel 배포 후 도메인에서 한 번 더 제출 테스트.
